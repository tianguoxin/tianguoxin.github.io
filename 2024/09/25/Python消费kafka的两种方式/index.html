Python消费kafka的两种方式

DaXin Space
2024-09-25
编程

之前一直用的python-kafka开发消费者的代码，但是在同一消费者组内并发消费topic时，它自动提交offset部分有问题，每次还需要手动的记录和维护，所以测试并记录以下两种方式。

第一种 基于faust的kafka消费方式

[[robinhood/faust: Python Stream Processing (github.com)]]
这东西准确来说有点像flink，主要作用是流式引擎。优点是安装起来比较简单，属于开箱即用，号称每天上亿数据没问题，并且支持并发，不用自己去考虑如offset提交等问题。缺点就是有一些笨重，再加上库的工作机理几乎是黑盒的

第二种 基于asyncio和aiokafka的协程式消费kafka

基于协程的并发也能提高消费kafka的速度，并且比多线程操作起来要容易一些
